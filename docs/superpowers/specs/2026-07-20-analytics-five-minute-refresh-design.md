# 运营大屏五分钟自动刷新设计

## 目标

运营大屏进入后立即加载数据，并在页面可见时每 5 分钟自动刷新。管理员仍可手动刷新，并能看到最近一次成功更新时间。

## 行为

- 自动刷新范围仅限 `/admin/analytics`，不影响游客洞察报告页。
- 页面初始化完成后立即加载一次数据，随后每 300000 毫秒检查并刷新。
- 浏览器标签页不可见时不发送自动刷新请求。
- 标签页重新可见时，如果距最近一次成功刷新已满 5 分钟，则立即刷新。
- 已有加载请求未结束时不重复发送请求。
- 景区或日期改变时沿用现有手动触发加载逻辑，并将成功时间作为新的刷新基准。
- 离开页面时清理定时器和 `visibilitychange` 监听器。
- 页面显示“每 5 分钟自动刷新”和最近一次成功更新时间；失败刷新不更新该时间。

## 实现边界

使用 Vue 页面生命周期、浏览器 Page Visibility API 和现有统计接口，不新增依赖，不引入 WebSocket，不修改后端接口。

## 验证

- 单元测试验证 5 分钟到期判断的边界。
- TypeScript/Vite 构建验证页面生命周期、模板和类型。
- 代码检查确认卸载清理、页面不可见跳过和并发保护均存在。
